# 一些提示词的使用

```plain
    请使用brainstorming 来分析下面的【功能或说明】，按以下步骤进行：
    0、分析出需要澄清的问题，如果没有澄清的问题，跳过【1】【2】，从【3】开始
    1、每个问题澄清时给出最优的2到3种解决方案，并记录下来，按顺序给出每个问题的3种解决方案
    2、所有问题澄清出3种方案后，请自动审核出最合适的一种方案组合
    3、选择出的最合适方案再次进行整体和大展观的进行审核，并由用户最终确认
    4、根据选择的方案组合 生成计划文档和实施计划，自动确认，不需要review
    5、使用sub agent(1) 来实现代码，无需要用户确认
    强制1：直接在当前分支上进行文档和代码生成，所有文件生成和修改完毕后 再由用户确认是否 git commit
```
